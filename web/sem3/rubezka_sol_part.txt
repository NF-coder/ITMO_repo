1.7 Какие преимущества можно выделить у принципа рендеринга CSR?

1. Уменьшения объема трафика, который необходим для взаимодействия с веб-сайтом, по сравнению с SSR, так как по сети после первоначальной загрузки передаются только
данные, связанные с динамическими операциями, в формате JSON
2. Проблема с поисковой индексацией в силу того, что поисковые роботы не исполняют JavaScript
3. Сравнительная “простота” реализации приложений на базе этого принципа, в силу того, что современные фреймворки широкого его поддерживают, и в силу того, что построение
приложений на подобных фреймворках “простое” с точки зрения технической реализации
4. Применение этого принципа снижает нагрузку на браузер клиента, так как всю работу по рендерингу страниц берет на себя сервер

Ответ: 1,3

const Button = ({symbol, onclick}) => {
	return (
		<button onClick={() => onclick()}>
			{symbol}
		<\button>
	)
} 

const Calculator = () => {
	const [getResult, setResult] = useState(0);
	const [getNum, setNum] = useState(0);

	const onklickNum = (num) => {
		return () => {
			setNum(getNum*10 + num);
		}
	}

	const onklickOp = (op) => {
		if (op === "+") return () => {
			setResult(getResult + getNum);
			setNum(0);
		}
		else if (op === "-") return () => {
			setResult(getResult - getNum);
			setNum(0);
		}
		else if (op === "*") return () => {
			setResult(getResult * getNum);
			setNum(0);
		}
		else if (op === "/") return () => {
			if (getNum === 0) {
				alert("Вы отчислены!")
				return;
			}
			setResult(getResult / getNum);
			setNum(0);
		}
		throw new Exception();
	}

	return (
		<>
			<table>
				<tr>
					<td><Button symbol={"1"} onclick={onklickNum(1)}><td/>
					<td><Button symbol={"2"} onclick={onklickNum(2)}><td/>
					<td><Button symbol={"3"} onclick={onklickNum(3)}><td/>
				</tr>
				<tr>
					<td><Button symbol={"4"} onclick={onklickNum(4)}><td/>
					<td><Button symbol={"5"} onclick={onklickNum(5)}><td/>
					<td><Button symbol={"6"} onclick={onklickNum(6)}><td/>
				</tr>
				<tr>
					<td><Button symbol={"7"} onclick={onklickNum(7)}><td/>
					<td><Button symbol={"8"} onclick={onklickNum(8)}><td/>
					<td><Button symbol={"9"} onclick={onklickNum(9)}><td/>
				</tr>
				<tr>
					<td colspan=3><Button symbol={"0"} onclick={onklickNum(0)}><td/>
				</tr>
				<tr>
					<td><Button symbol={"+"} onclick={onklickOp("+")}><td/>
					<td><Button symbol={"+"} onclick={onklickOp("-")}><td/>
					<td><Button symbol={"+"} onclick={onklickOp("*")}><td/>
				</tr>
				<tr>
					<td><Button symbol={"+"} onclick={onklickOp("/")}><td/>
					<td colspan=2><Button symbol={"="} onclick={onklickOp("=")}><td/>
				</tr>
			</table>
			<p>Результат: {getResult}</p>
		</>
	)
}

2.6. Какие из приведенных ниже утверждений о спецификации JAX-RS верны?  

1. JAX-RS обязывает разработчика использовать подход REST к построению API  
2. JAX-RS не поддерживает работу с файлами, для работы с ними необходимо реализовывать отдельный сервис  
3. JAX-RS позволяет реализовать API байкера, приложения без написания HTTP ответов вручную разработчиков  
4. В большинстве реализаций спецификации JAX-RS по умолчанию используется XML в качестве формата тела запроса и тела ответа
